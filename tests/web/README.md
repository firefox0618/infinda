# Web Tests

Каталог `tests/web` предназначен для e2e и пользовательских frontend-сценариев.

Сюда должны попадать тесты, которые:
- проходят через реальные маршруты `apps/web`;
- проверяют связку BFF/frontend/backend на уровне пользовательского поведения;
- не сводятся к одному локальному компоненту.

Примеры будущих сценариев:
- вход и выход из кабинета;
- отправка сообщения в поддержку из кабинета;
- привязка Telegram через профиль пользователя.

Текущие реальные сценарии:
- `test_auth_to_support.spec.ts` — регистрация пользователя, переход в кабинет и отправка сообщения в поддержку через UI.
- `test_logout.spec.ts` — регистрация пользователя, вход в кабинет, logout и возврат на `/auth`.
- `test_login.spec.ts` — регистрация, logout и повторный вход в тот же аккаунт.
- `test_telegram_link.spec.ts` — открытие профиля и создание deep-link для привязки Telegram.
- `test_subscription_renew.spec.ts` — открытие модалки продления и проверка каталога тарифов.

Особенность запуска:
- `start_e2e_stack.sh` перед стартом backend автоматически выполняет `manage.py migrate --noinput`, чтобы e2e не зависел от ручной подготовки локальной базы.
- frontend для e2e поднимается через `next build` и `next start`, а не через `next dev`, чтобы пользовательский сценарий проверялся на стабильном production-like рантайме.
- `run_e2e.sh` поднимает backend и frontend, ждет health-check обоих сервисов и только после этого запускает Playwright.
