# Tests

Каталог `tests/` зарезервирован под проверки уровня системы.

Текущее состояние:
- backend app-level тесты уже живут внутри `apps/api/apps/*/test_*.py`;
- frontend app-level тесты пока не развернуты;
- корневой интеграционный и e2e-слой уже начал наполняться реальными сценариями;
- базовая структура `tests/api` и `tests/web` уже заведена;
- в `tests/api` уже есть несколько сквозных backend-сценариев;
- в `tests/web` уже есть первый реальный Playwright-сценарий `test_auth_to_support.spec.ts`.

Планируемое наполнение:
- `tests/api` — интеграционные проверки поверх нескольких backend-доменов;
- `tests/web` — e2e и сценарии пользовательских маршрутов frontend;
- общие системные фикстуры и служебные сценарии уровня репозитория.

Пока этот каталог еще нельзя считать полным источником фактического покрытия, но он уже используется для реальных интеграционных и e2e-проверок.
