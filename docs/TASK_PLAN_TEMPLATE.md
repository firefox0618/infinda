# TASK PLAN TEMPLATE

Шаблон используется перед каждой реализацией, чтобы этап разработки оформлялся в одном формате:

`контекст -> план -> файлы -> проверки -> риски`

## 1. Контекст

- Что нужно сделать.
- Зачем это делается.
- Что уже есть в проекте.
- Какие ограничения и договоренности важны для этой задачи.
- Какие HTML-макеты являются визуальным эталоном, если задача затрагивает интерфейс.

## 2. Цель этапа

- Один короткий и понятный результат текущего этапа.
- Только то, что действительно должно быть сделано сейчас.

Пример:
- Подготовить каркас страницы авторизации без подключения реального API.

## 3. План

Указать только шаги текущего согласованного этапа.

Пример:
1. Подготовить структуру страницы и layout.
2. Вынести форму входа в отдельный компонент.
3. Вынести типы формы в отдельный файл.
4. Подготовить заглушку сервиса авторизации.

## 4. Файлы

### Будут созданы

- `path/to/new-file`
- `path/to/another-file`

### Будут изменены

- `path/to/existing-file`

### Не должны быть затронуты

- `path/to/untouched-file`

## 5. Архитектурные вопросы

### Уже согласовано

- ...

### Требует согласования

- ...

### Нельзя решать молча

- смена стека;
- новые зависимости;
- изменение структуры данных;
- изменение архитектурного слоя;
- изменение визуального стиля эталонных HTML-макетов.

## 6. Проверки

### Планируемые проверки

- форматирование;
- линтер;
- type-check;
- тесты;
- сборка;
- ручная проверка, если возможна.

### Если что-то пока невозможно

- Что именно не получится проверить.
- Почему это пока невозможно.

- Что зависит от следующего этапа.
- Какие места потребуют дополнительной проверки.
