# RULES

Этот документ больше не дублирует полный список правил.

Единый источник постоянных правил проекта:
- [AGENTS.md](../AGENTS.md)

Зачем это сделано:
- чтобы правила не расходились между несколькими файлами;
- чтобы у проекта был один актуальный источник процесса и инженерных ограничений;
- чтобы при обновлении правил не приходилось синхронизировать несколько копий вручную.

Как использовать документы:
- `AGENTS.md` — главный документ с постоянными правилами работы.
- `README.md` — точка входа в проект и навигация по документации.
- `docs/MEMORY.md` — рабочий контекст и текущие договоренности.
- `docs/PROJECT_STRUCTURE.md` — структура проекта и назначение каталогов.
- `docs/PHASE_01_FOUNDATION.md` — рамки и результат текущего этапа.

Правило поддержки актуальности:
- если меняются постоянные правила процесса, сначала обновляется `AGENTS.md`;
- остальные документы должны ссылаться на него, а не переписывать те же правила заново.
