# PHASE 01: FOUNDATION

Цель этапа:
- Подготовить понятный каркас проекта до начала основной разработки.

Что входит в этап:
- Зафиксировать структуру каталогов.
- Зафиксировать правила разработки.
- Зафиксировать память проекта и договоренности.
- Подготовить документацию для следующих этапов.
- Отделить визуальные макеты от будущей продуктовой реализации.

Что не входит в этап:
- Полная реализация frontend.
- Полная реализация backend.
- Интеграции с оплатой, Telegram, 3X-UI.
- Реальные API и бизнес-логика.

Результат этапа:
- Есть понятный каркас проекта.
- Есть единые правила работы.
- Есть точка отсчета для последующей реализации.
- Есть место для frontend, backend, общих пакетов, тестов и инфраструктуры.
- Выбран и зафиксирован стек: `Next.js` и `Django + DRF`.
- В `apps/web` создан первый переиспользуемый frontend-каркас и перенесена страница `главная`.

Статус документа:
- Этот документ фиксирует рамки исходного этапа foundation.
- Фактическое состояние репозитория уже вышло за пределы только foundation: в коде есть несколько frontend-страниц, BFF/proxy-слой, backend-домены и API-тесты.
- Актуальное текущее состояние проекта нужно смотреть в `README.md` и `docs/MEMORY.md`.
- Документ нужно воспринимать как архивную точку отсчета, а не как актуальный чеклист текущей реализации.

Следующие вопросы на согласование:
1. Когда разворачиваем корневой интеграционный и e2e-слой в `tests/`.
2. Когда наполняем `infra` реальными окружениями, proxy и CI/CD-конфигами.
3. В каком порядке декомпозируем крупные UI-first модули вроде `cabinet`.
